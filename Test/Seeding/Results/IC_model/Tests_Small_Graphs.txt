Here we execute both Basic Greedy and Voterank, on 5 different triples of full, partial and recovered graphs.
For each triple, both algorithms are run 20 times for each of the three variants, plus the "real" run where they're
executed on the full graph again, but with the restriction that "virtual" nodes (i.e. new nodes created by the
InfluentialNodeRecovery algorithm) cannot be selected as seeds.
All of the runs are repeated for different k (max. number of seeds) and the mean over the 20 runs is shown for each of the
8 cases.

The above test is run two times:
- with 50 hidden nodes from the initial graph;
- with 100 hidden nodes.

The graph parameters are shown before each test.
The graphs selected here are "small", i.e. with only 300 nodes for computational purposes: as execution times show
after each table, Basic Greedy is particularly prohibitive and its complexity grows exponentially with the dimension
of the graph.

For this reason, the following tests, that use graphs with more nodes and edges, will be run exclusively on Voterank,
here proven faster and more efficient.
